What This Is

AI Origins Network is a minimal entry into a distributed observation layer. A node is not manually provisioned as an artifact. It is initiated, identified, and made visible through live network behavior.

Lightweight entry

No image distribution. No heavy setup. A compatible machine can join directly through a single public install path.

Observable presence

After joining, the node begins emitting heartbeat signals and becomes visible through the public presence surface.

What Joining Means

  • A machine becomes a visible node in the observation network
  • A node receives a local identity and begins live continuity signaling
  • The system remains minimal, reproducible, and directly deployable
  • Presence is public; certification and issuance remain separate layers

Node Types

Core Node

Aggregation, coordination, and structural continuity.

Edge Node

Execution, observation, and distributed network participation.

External Node

Partner, affiliate, or third-party machine joining the public observation layer.

Presence Layer

The public visibility surface through which nodes can be observed while active.

Install

Run the public bootkit installer on any compatible Debian or Armbian machine:

curl -sSL https://aiorigins.org/bootkit/install.sh | bash
Current release: v1.3
Public install path has been moved to the domain entry.
Enrollment and network-side behavior remain operator-controlled at this stage.

What Execution Does

  • Generate a local node identity
  • Attempt registration into the observation layer
  • Start heartbeat automatically
  • Make the node visible through the network presence surface when active

Requirements

  • Debian or Armbian compatible machine
  • Network connection available
  • Basic shell access
  • Any suitable machine may become a node

Node Behavior

  • Each node generates a unique local identity at deployment time
  • Heartbeat establishes observable continuity
  • Presence data is emitted to the network layer
  • Nodes are not merely deployed. They emerge as visible participants

Status

Nodes are visible through the public network presence layer. This is the live observation surface of the network, where active nodes can be seen as present entities.

After Joining

Joining the observation network does not by itself create a certificate or audit statement. It establishes presence first.

  • Your node receives an active local identity
  • Your machine becomes part of the distributed observation surface
  • Heartbeat continuity makes the node publicly visible when online
  • Evidence, signatures, and formal issuance belong to later layers
