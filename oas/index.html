Origin Audit Standard (OAS) · v0.1

OAS is a provenance-style integrity snapshot format designed for repeatability and third-party verification. It is not a security rating and does not claim vulnerability findings.
OAS v0.1 is a reproducible proof package format. It produces verifiable snapshots, not vulnerability claims.

Public sample report (target: iana.org)

  • Latest sample manifest (/oas/sample/latest/manifest.json)
  • Fixed sample manifest (/oas/sample/AIR-2026-03-02T11:45:49Z/manifest.json)
The sample demonstrates: canonical self-hash manifest and a chain-ready anchor reference (internal). Artifacts are minimal in test mode.

Verify in browser (no upload)

Open /oas/verify/, then upload manifest.json. Verification runs locally in your browser: it recomputes the canonical self-hash and compares it with manifest.anchor.current_hash.

Verify offline (CLI)

Download the sample package and run:
python3 verify_oas.py --manifest manifest.json
If an anchor log is provided, you can also run: 
python3 verify_oas.py --manifest manifest.json --anchor-log anchor.log
