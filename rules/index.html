AI Origins Node Rules

Admission and participation policy for nodes joining the AI Origins network.

Core principle

The AI Origins network is a specification network. Hardware vendors, hosting method, and operating systems may differ, but node identity, protocol compatibility, endpoint correctness, and admission policy must follow the published rules.

Node categories

  • CORE – root governance and identity anchor.
  • FIXED – stable production nodes.
  • MOBILE – portable presence nodes.
  • RESERVE – standby infrastructure.
  • EDGE / SERVICE – public or limited-scope service nodes.

Admission levels

  • Self-service: edge, witness, mobile
  • Review required: service / fixed
  • Invitation only: backbone / quorum / core-adjacent

Minimum requirements

  • Unique node identifier
  • Valid public key
  • Reachable endpoint
  • Machine readable node.json
  • Agreement with audit and abuse handling policy

Operational rule

Existing audit-era files should remain untouched unless a change is required. Stable paths are preserved first; structural optimisation can happen later.
