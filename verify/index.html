Join Verification

Verification interface for nodes requesting admission to the AI Origins network.

Join Check API

POST /api/join-check
Content-Type: application/json

{
 "network":"AI Origins",
 "policy_version":"Join Node Spec v1.0",
 "node_id":"fixed-qingdao-01",
 "node_role":"fixed",
 "region":"CN-QD",
 "endpoint":"qingdao.example.net:51820",
 "public_key":"BASE64_PUBLIC_KEY",
 "metadata_url":"https://example.org/node.json"
}

Expected response

{
 "ok": true,
 "status": "PASS",
 "checks": {
   "metadata": "PASS",
   "public_key": "PASS",
   "endpoint": "WARN",
   "policy": "PASS"
 }
}

Status meaning

  • PASS – node meets protocol rules.
  • WARN – minor issues but acceptable.
  • FAIL – node incompatible with policy.

Typical verification flow

prepare node.json
→ deploy node
→ run join-check
→ PASS / WARN / FAIL
→ admission review
→ node activated
